Important Notes

Telemetry and privacy

EasyKit supports opt-in telemetry. Data is limited to anonymous usage and error aggregates. To opt-out: Settings → Privacy → Telemetry → Disable.

Logging & Diagnostics

Logs are written to the application data folder. When reporting issues, attach the latest log bundle. For verbose logging enable Settings → Diagnostics → Logging Level → Debug. See repository helpers for log path utilities in the source tree.

Configuration

Use configuration profiles to manage different environments (development, CI, production). Tool paths can be set per-user or left to auto-detection; prefer system PATH installs for predictable behavior.

Security & Vulnerability Reporting

Report security issues privately via the repository's security contact or private advisory channel. Provide reproduction steps, affected versions, and logs. Do not disclose sensitive issues publicly until patched.

Release & Versioning

EasyKit follows semantic versioning. Check the changelog and releases page before upgrading across major versions—breaking changes to plugin schemas require a major bump and may require tool manifest updates.

Marketplace authors

Tool authors should provide clear manifests, documentation, and minimal required permissions for their tools. Use pre/post hooks sparingly and document expected side effects.
